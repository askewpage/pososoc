# 🚨 Log Viewer from Helll

Аркадный симулятор SOC-аналитика. Бесконечная лента системных логов сыплется на экран — среди тысяч `[INFO]` спрятаны реальные инциденты (`[CRITICAL]`, но не только). Успей поймать угрозу кликом или клавишей `Enter`/`Space`, прежде чем строка «утечёт» с экрана.

Чистый HTML/CSS/JS, без сборки и зависимостей — открывается прямо из `index.html` и разворачивается на GitHub Pages за одну кнопку.

## Режимы игры

| Режим | Описание |
|---|---|
| 🎓 **Training** | Угрозы подсвечиваются цветом по категории (легенда сбоку). Жизни бесконечны — учимся распознавать паттерны атак без риска. |
| 🕵️ **Standard** | Без подсказок. Прогрессия уровней (Trainee → Cyber Sentinel), 3 жизни за пропущенные инциденты, шкала стресса (выгорание аналитика). |
| ⚡ **Turbo** | Логи летят так быстро, что читать некогда — лови по ключевым словам (`401`, `SQL Injection`, `rm -rf`...). Сложность растёт со временем выживания. |

## Фишки из реальной работы SOC

- **Эскалация тикетов** — после поимки угрозы всплывает карточка классификации: угадай категорию атаки (SQLi, XSS, brute force...) за 4 секунды ради бонусных очков и тега MITRE ATT&CK.
- **Очередь тикетов** — боковая панель показывает последние эскалации со статусами `PENDING` / `CONFIRMED` / `MISSED`.
- **Threat Intel Feed** — декоративная лента "живых" IOC (вредоносные IP, хэши, домены C2).
- **Alert fatigue** — шкала стресса растёт от ложных срабатываний и пропущенных инцидентов; на 100% — выгорание и конец смены.
- **Маскировка угроз** — на высоких уровнях реальные атаки иногда логируются как `INFO`/`WARNING`, а не `CRITICAL` (как бывает в жизни).
- **16 категорий уязвимостей**: brute force, SQL Injection, XSS, path traversal, command injection, port scanning, malware, phishing, privilege escalation, exfiltration, impossible travel, ransomware, C2 beaconing, DDoS, insider threat, 0-day/CVE.

## Геймификация

- Очки, комбо-множитель, лучшая серия.
- 8 уровней сложности в Standard-режиме (растёт скорость и плотность логов, включаются новые категории угроз).
- 13 достижений (First Blood, SQL Slayer, Turbo Master, SOC Legend...), сохраняются локально.
- Локальная таблица рекордов — отдельная для каждого режима.
- Field Manual — справочник по всем категориям уязвимостей с примерами строк лога.

## Управление

- Клик мышью по строке лога — поднять тикет.
- `Enter` / `Space` — поймать самую новую строку (удобно в Turbo-режиме).
- `Esc` / `P` — пауза.

## Запуск локально

Модули загружаются через `<script type="module">`, поэтому открыть `index.html` напрямую через `file://` не выйдет (браузеры блокируют модули по CORS) — нужен локальный HTTP-сервер:

```bash
# любой вариант:
python3 -m http.server 8080
# или
npx serve .
```

Затем открыть `http://localhost:8080`.

## Деплой на GitHub Pages

Репозиторий уже готов к загрузке — файлы лежат в корне, сборка не требуется.

**Вариант 1 — без Actions (проще всего):**
1. Создать репозиторий на GitHub и запушить содержимое этой папки в ветку `main`.
2. Settings → Pages → Source → **Deploy from a branch** → ветка `main`, папка `/ (root)`.
3. Через минуту сайт будет доступен по адресу `https://<username>.github.io/<repo>/`.

**Вариант 2 — через GitHub Actions (уже настроено):**
В репозитории есть `.github/workflows/pages.yml`. Достаточно:
1. Settings → Pages → Source → **GitHub Actions**.
2. Запушить в `main` — workflow соберёт и задеплоит сайт автоматически.

## Структура проекта

```
index.html            разметка страницы
css/style.css          вся стилизация (тёмная тема, анимации, адаптив)
js/data.js             шаблоны логов, категории уязвимостей, кривая сложности
js/game.js             игровой движок (спавн строк, счёт, жизни, стресс)
js/audio.js            процедурные звуки через WebAudio (без бинарных файлов)
js/storage.js          localStorage: рекорды, достижения, настройки
js/achievements.js     список достижений и условия разблокировки
js/main.js             UI-склейка: экраны, HUD, модалки
.github/workflows/     автодеплой на GitHub Pages
```

Никаких сборщиков, фреймворков или внешних CDN — всё самодостаточно и будет работать даже без интернета после первой загрузки страницы.
